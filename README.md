# 스도쿠 솔버 (Sudoku Solver)

C++ Win32 API 기반 네이티브 Windows GUI 스도쿠 해결 프로그램입니다.

## 실행 방법

[Releases](https://github.com/needitem/Sudoku-Solver/releases)에서 `sudoku.exe` 다운로드 후 실행

## 해결 알고리즘

### 1. 비트마스킹 기반 후보 관리

각 행, 열, 3x3 박스마다 9비트 마스크를 사용하여 이미 사용된 숫자를 추적합니다.

- 비트 0 = 숫자 1, 비트 1 = 숫자 2, ... 비트 8 = 숫자 9
- 특정 셀의 후보 = `~(행마스크 | 열마스크 | 박스마스크) & 0x1FF`

이 방식은 set 자료구조 대비 메모리와 연산 속도 모두 효율적입니다.

### 2. Naked Singles (단일 후보)

셀에 가능한 숫자가 딱 하나만 남은 경우 해당 숫자로 확정합니다.

예시:
```
┌───────┬───────┬───────┐
│ 5 3 . │ . 7 . │ . . . │
│ 6 . . │ 1 9 5 │ . . . │
│ . 9 8 │ . . . │ . 6 . │
└───────┴───────┴───────┘
```
(0,2) 위치: 행에 5,3,7 사용, 열에 8 사용, 박스에 5,3,6,9,8 사용
→ 후보 = {1,2,4} 중 하나가 아닌 경우, 유일한 후보로 확정

### 3. Hidden Singles (숨은 단일)

특정 숫자가 행/열/박스 내에서 한 곳에만 들어갈 수 있는 경우를 찾습니다.

예시:
```
행 0: [5, 3, _, _, 7, _, _, _, _]
```
숫자 4가 이 행에서 (0,5) 위치에만 들어갈 수 있다면 → (0,5) = 4로 확정

### 4. Backtracking (백트래킹)

논리적 방법으로 해결되지 않으면 시행착오 방식을 사용합니다.

1. 후보가 가장 적은 빈 셀 선택 (MRV 휴리스틱)
2. 가능한 숫자 하나씩 시도
3. 모순 발생 시 되돌리고 다음 숫자 시도
4. 모든 셀이 채워지면 성공

## 해결 과정 요약

```
입력 → Naked Singles 반복 → Hidden Singles 반복 → 진전 없으면 Backtracking
         ↑___________________________________|
```

## 최적화 기법

| 기법 | 설명 |
|------|------|
| 비트마스킹 | O(1) 후보 계산, 메모리 효율적 |
| MRV | 후보가 적은 셀 우선 선택으로 탐색 공간 축소 |
| 조기 종료 | 후보가 0개인 셀 발견 시 즉시 백트랙 |

## 사용법

1. 프로그램 실행
2. 초기 숫자 입력 (1-9)
3. "해결" 버튼 클릭
4. 해결된 숫자는 파란색으로 표시
5. "초기화"로 리셋

## 빌드 (선택)

Visual Studio Developer Command Prompt에서:
```cmd
cl /EHsc /O2 /utf-8 sudoku.cpp user32.lib gdi32.lib comctl32.lib /Fe:sudoku.exe
```
